pub struct FuncBuilder<'a, TVar> {
    pub editor: FuncEditor<'a>,
    sealed_bbs: BitSet,
    filled_bbs: BitSet,
    variable_map: BTreeMap<TVar, (Ty, BTreeMap<BBId, Index>)>,
    incomplete_phi: BTreeMap<BBId, Vec<(TVar, Index)>>,
    pub block_pred: BTreeMap<BBId, TinyVec<[BBId; 7]>>,
}

A function builder that loosely resembles building SSA functions using the algorithm described in Simple and Efficient Construction of Static Single Assignment Form by Matthias Braun et al.

This type is parameterized by one type, TVar, which is the representation of variable in your original language. It is expected to be a small, Clone Ord and Debug type for usage inside BTreeMaps

Fields

editor: FuncEditor<'a>

The function we're building.

sealed_bbs: BitSet

Sealed basic blocks.

Sealed basic blocks have all their predecessors determined.

filled_bbs: BitSet

Filled basic blocks.

Filled basic blocks have finished filling in calculation instructions.

variable_map: BTreeMap<TVar, (Ty, BTreeMap<BBId, Index>)>

A Variable - Basic Block - SSA Value map.

This map is for global and local value numbering in the algorithm. Since we expect variables and basic blocks to be small integer IDs, BTreeMap is preferred here over HashMap.

incomplete_phi: BTreeMap<BBId, Vec<(TVar, Index)>>

Incomplete phi commands (params in our case).

Basic blocks predecessors

Set current basic block to bb_id. Also sets current_idx to the sentinel value that moves backward to the last instruction, and forward to the first instruction.

Returns whether the position was unchanged.

Note

The reason why this function sets the position at a sentinel value is to ensure one can use [move_forward] as the condition of a while loop to visit all instructions in this function.
